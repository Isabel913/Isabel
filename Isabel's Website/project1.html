Project 1

Representation

System Overview

My topic is a shooting bird game. I made it a game because I used to play a lot of games when I was little, and games usually present the feeling of winning and rewarding, plus a little bit of anxious, which would help engage the users.

Ideation Diagram

In the ideation phase, I imagined the system to contain a lot of elements like sounds, music, instructions, gameover, etc.

Revised Diagram

But in the end, I restructured the whole system and keeping only the necessary elements to keep the users engaged, and not overwhelmed.

Translate Diagram into Code

I coded my diagram with processing, added images for background, bird, and arrow, following my diagram, reconstructed a bit and coded the interactive elements.

Interactive Elements

My interactive elements are : Move the position of the arrow container with “Right” and “Left” keys on the keyboard, tap “Space” bar if users want to shoot an arrow, and adding scored to both Birds and User depending on if the bird got killed or escaped. I decided to use keyboards because tapping keyboard is an easy interaction for the users comparing to clicking or dragging using trackpad / mouse.

Obstables

My biggest obstacle was making sure when the arrow shoot the bird, the bird disappears, as well as the scoring part, as shown in the code. I had to watch a few tutorials to figure that out.

Besides that, everything ran smoothly.
